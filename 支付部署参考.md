## 一、API 域名架构

| 用途 | 域名 | 平台 |
|------|------|------|
| 前端页面 | `gk.minicode.cloud` | Cloudflare Pages |
| API + 回调 | `gk.minicode.cloud/api/*` | Pages Functions（同域名） |
| 支付回调地址 | `https://gk.minicode.cloud/api/callback` | Pages Functions |

**注意：** API 不走单独的子域名（如 `gk-api.minicode.cloud`），避免跨域问题。前端用 `apiPrefix = "/api"` 同域名调用。

## 四、支付流程

```
用户点击购买
    ↓
前端 POST /api/createorder
    ├─ 数据库创建订单（status=0）
    ├─ 分配卡密（标记is_used=1）
    ├─ 调用虎皮椒 payment/do.html
    └─ 返回 {payUrl, tradeNo}
    ↓
前端展示二维码 → 用户手机扫码支付
    ↓
┌─ 虎皮椒回调 POST /api/callback ──────────────┐
│  参数: form表单 (status=OD 表示已支付)          │
│  验证签名 → 更新订单 status=1 → 返回 "success"  │
└──────────────────────────────────────────────┘
    ↓
前端轮询 /api/order/{tradeNo} (每3秒)
  或用户点击"确认" → 调用 /api/verify-payment
    ↓
订单 status=1 → 展示网盘链接 + 提取码
```

## 五、回调参数对照（关键！）

虎皮椒回调 POST 到 `notify_url` 时，参数为 **form表单格式**，关键字段：

| 参数字段 | 值示例 | 说明 |
|----------|--------|------|
| `trade_order_id` | `GK1234567890` | 商户订单号 |
| `status` | `OD` | **OD=已支付**, WP=待支付, CD=已取消 |
| `total_fee` | `1.99` | 支付金额 |
| `transaction_id` | 微信交易号 | 微信内部订单号 |
| `open_order_id` | 虎皮椒内部单号 | 虎皮椒内部订单号 |
| `attach` | 自定义备注 | 创建订单时传入的 attach 字段 |
| `hash` | MD5签名 | 验签用 |

**历史踩坑：** 之前错误地用了 `trade_status === 'TRADE_SUCCESS'` 判断支付状态，正确字段是 **`status === 'OD'`**。

## 六、查询API参数对照

主动查询虎皮椒支付状态：

| 参数名 | 必填 | 说明 |
|--------|------|------|
| `appid` | 是 | 虎皮椒APPID |
| `out_trade_order` | 是 | **商户订单号**（注意不是 trade_order_id） |
| `time` | 是 | 当前时间戳（秒） |
| `nonce_str` | 是 | 随机字符串 |
| `hash` | 是 | MD5签名 |

响应：
```json
{
  "errcode": 0,
  "data": { "status": "OD" },  // OD=已支付, WP=待支付
  "errmsg": "success!"
}
```

**历史踩坑：** 之前用了 `trade_order_id` 参数名，正确为 **`out_trade_order`**；且缺了 `time` 和 `nonce_str` 必填字段。

## 七、签名算法

所有请求虎皮椒 API 的参数都需要生成 MD5 签名：

```
1. 剔除 hash 参数本身
2. 非空参数按 ASCII 升序排序
3. 拼接为 key1=value1&key2=value2 格式
4. 末尾拼接 AppSecret（之间无任何连接符）
5. 对整串做 MD5，取 32 位小写
```

PHP 示例（官方）：
```php
function generate_xh_hash(array $datas, $hashkey) {
    ksort($datas);
    $arg = '';
    foreach ($datas as $key => $val) {
        if ($key == 'hash' || is_null($val) || $val === '') continue;
        $arg .= ($arg ? '&' : '') . "$key=$val";
    }
    return md5($arg . $hashkey);
}
```

## 九、数据库 D1

```
数据库名: gk-db
Database ID: 7e9e2bc1-cc6c-4889-8bcc-a697961559d4
```

核心表：`goods`（商品）、`card`（卡密→网盘链接）、`orders`（订单）、`users`（用户）、`members`（会员）

```bash
npx wrangler d1 execute gk-db --command="SELECT * FROM orders ORDER BY create_at DESC LIMIT 10" --remote
```

手动标记支付：
```bash
npx wrangler d1 execute gk-db --command="UPDATE orders SET status=1,pay_at=CURRENT_TIMESTAMP WHERE trade_id='GKxxxx'" --remote
```
